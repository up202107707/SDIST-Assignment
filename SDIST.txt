Setup Instructions (Add to Your Environment)
Before running the script, set up a LAN testbed (e.g., two Linux machines: one as server, one as client running the script). References from the proposal (e.g., Andr√© Perez's book) describe PTP/NTP mechanics in detail.
NTP Setup:

Server (e.g., Ubuntu machine on LAN): Install ntp (sudo apt install ntp). Edit /etc/ntp.conf to include server 0.pool.ntp.org (upstream) and restrict 192.168.1.0 mask 255.255.255.0 nomodify notrap (allow LAN clients). Restart: sudo systemctl restart ntp.
Client (machine running script): Install ntp if needed, but since script uses custom Python client, no config required. Point script's ntp_server to server's IP (e.g., '192.168.1.10').

PTP Setup (IEEE 1588):

Install linuxptp on both machines: sudo apt install linuxptp.
Server: Run as master on an Ethernet interface (e.g., eth0): sudo ptp4l -i eth0 -m. Optionally sync system clock: sudo phc2sys -s eth0 -O 0.
Client: Run as slave: sudo ptp4l -i eth0 -s -m. Sync system clock: sudo phc2sys -c eth0 -s CLOCK_REALTIME -O 0. The script queries via pmc, so ensure sudo works (or run script with sudo).
Note: PTP requires direct Ethernet (no WiFi/router interference for sub-ms accuracy). Test with pmc command manually.

Run the script on the client machine. For parameter variation, adjust the param_variations list in main().
